# 论文写作项目（Agents 驾驶舱）

本文件是**论文项目的智能体驾驶舱与技能路由正文**：复制到具体课题仓库根目录后，按「项目信息」「目录占位符」逐项填写；与 [README.md](README.md) 配合使用（README 侧重 **thesis skills** 技能包说明与安装，**本文件侧重「你的课题怎么跑」**）。文中举例**轮换学科与排版**，仅为占位，不代表维护者领域。

技能入口统一为仓库根目录 **`skills/<技能名>/SKILL.md`**（若 Cursor/OpenCode 约定 `.cursor/skills/` 等路径，请先按 [README.md](README.md) 复制或链接，再让 Agent 读取）。

---

## 0. 初始化询问（Agent 必读）

### 何时视为「未初始化」

满足**任一**即可判定为未初始化，执行论文类任务前应先完成下文询问（或用户显式跳过）：

- 本节下方「**初始化状态**」为 **未初始化**，或留空；
- 第 1 节表格里**论文题目、学科与方向、当前阶段**仍为 `_待填_` 等明显占位；
- 第 2 节占位符尚未对应到**实际路径或明确说明**（如「暂无代码目录」）。

**可不触发初始化询问的情况**：用户仅咨询技能包结构、某 `SKILL.md` 含义、仓库安装步骤；或用户**明确说**「跳过初始化，直接……」并给出当次任务所需最小上下文。

### Agent 在未初始化时的行为

1. **不要**直接开始润色、代写章节、选技能执行长流程；**先**用「初始化问题清单」发起一轮询问（可合并为一条消息）。  
2. 用户回答后，再选技能、读 `SKILL.md`、执行任务。  
3. 若用户同意写入本文件：仅将确认过的信息填回第 1–2 节，并把「初始化状态」改为 **已初始化**；**不得**在用户未同意时大幅改写本文件其他章节。

### 初始化问题清单（可直接发问）

按项目实际情况删减提问，建议至少覆盖加粗项：

1. **论文题目**（中或英，或暂拟题）  
2. **学位类型**（学硕 / 专硕 / 博士等）与**当前阶段**（开题 / 初稿 / 盲审前 / 答辩前等）  
3. **学科与方向**（便于术语与文献策略）  
4. **正文语言**与**排版**（如 LaTeX、Word 或二者混用；若已知请说明主 `.tex` / `.docx` 大致位置）  
5. **参考文献习惯**（如 BibTeX + Zotero / 手工等）  
6. **正文所在路径**（对应 `{{THESIS_DIR}}`）、**润色/对照稿路径**（对应 `{{DRAFT_DIR}}`，若没有可说「暂无」）  
7. 是否有**代码/实验仓库路径**（`{{CODE_DIR}}`）、**图件目录**（`{{FIGURES_DIR}}`）——没有可答「无」  
8. **当次对话最想完成的一件事**（一句）

用户答完后，可将「初始化状态」更新为 **已初始化**（见下表）。

## 2. 目录与占位符

将下列路径改为你仓库中的**真实目录名**，并在对话里沿用同一说法（子 Agent 与 Cursor 规则中的占位符与此一致）。

| 占位符 | 默认示例 | 用途 |
|--------|----------|------|
| `{{THESIS_DIR}}` | `论文章节/` | 学位论文正文、参考文献主文件（如 `.tex` / `.bib`） |
| `{{DRAFT_DIR}}` | `my-thesis/` | 润色对照稿、大段改写备份、非最终定稿 |
| `{{CODE_DIR}}` | 依项目自定，如 `code/`、`experiments/` | 实验与复现代码（**论文域不默认改这里**） |
| `{{FIGURES_DIR}}` | 依项目自定，如 `figures/`、`plots/` | 制图产出与源文件（**论文域不默认改这里**） |

**本仓库若未创建上述目录**：可先在占位符表写「计划路径」，成稿前再建目录；或保持本技能包纯文档用法，仅填写你实际存在的路径。

## 3. 与子 Agent 协作（多智能体路由）

域内细则见 `.cursor/agents/`。主 Agent 可按**当前编辑文件路径**或任务类型委派，也可由你显式调用：

| 目录/场景 | 子 Agent | 调用示例 |
|-----------|----------|----------|
| `{{THESIS_DIR}}`、`{{DRAFT_DIR}}` 内正文、参考文献、结构 | thesis-writing-agent | `/thesis-writing-agent` |
| `{{CODE_DIR}}` 内实验、求解、复现脚本 | engineering-code-agent | `/engineering-code-agent` |
| `{{FIGURES_DIR}}` 内制图、可视化、出图配置 | mapping-agent | `/mapping-agent` |

**跨域约定**

- 不在论文域、代码域、制图域之间**顺手改**对方文件；确需联动时先列出「变更影响清单 / 对齐清单」。
- 论文 ↔ 代码 ↔ 图表不一致时，以**可复现结果**为准；修订论文表述并保留对照稿。
- **流程图**：论文叙事（方法链、算法或论证结构）由 thesis-writing-agent 主责排版与表述；事实与代码/制图侧校对。

## 4. 技能使用原则

0. **初始化**：见第 0 节。判定为未初始化且用户未声明跳过时，**先完成初始化询问**，再选技能；不要跳过询问直接承接全文润色、代写章节等论文主任务。
1. **先盘点**：任务是否已有对应 `skills/<名>/SKILL.md`；新类型任务先给出技能方案，**你确认后再**新建技能（可借助 `skill-creator`）。
2. **有则必读**：执行前用 Read 读取对应 `SKILL.md`，按其中流程与约束操作，不得仅凭技能名口头发挥。
3. **同一任务只选一个技能**，避免混用（尤其文献综述「述」与末章「展望」、摘要与全文评审等易重叠场景）。
4. **长期约定**（如把个人偏好写入本文件）：若希望 Agent 写入或大幅改写 `agents.md`，请先征得你同意。

## 6. 文献类任务推荐路径（组合示例）

以下按「阶段」给出**常见顺序**，仍须遵守「一步一技能、先读 SKILL.md」：

- **学位论文文献综述（只要「述」，不要末章展望）**  
  `literature` → `literature-summary-zh` →（润色）`academic-writing` →（可选）`thesis-reviewer`

- **系统综述 / PRISMA**  
  `systematic-review-lite` →（流程图）`svg-flowchart` →（正文段落）`literature-summary-zh` 或 `academic-writing`

- **SCI / 顶会定向检索**  
  `sci-journal-literature` 与 `literature` 分工；题录统一进 BibTeX 时用 `literature` 中的工具与约定

## 7. 示例开场（可粘贴到对话）

**首次协作、尚未在 agents.md 中填写项目信息时**，可先发：

```text
请先阅读 agents.md 第 0 节：我尚未完成初始化，请按「初始化问题清单」问我一轮，
再根据我的回答选技能并执行。
```

**已初始化或本次已说明上下文时**，可发（学科与工具请按实际替换；勿拘泥于下例用词）：

```text
我在写【学硕/博士】论文，题目是「……」，正文在「{{THESIS_DIR}}」目录（请替换为实际路径）。
排版为【LaTeX + BibTeX / 仅 Word / 混用】之一。请先根据本仓库 skills 与 agents.md 选择**唯一**技能，
读取对应 SKILL.md 后再执行：【具体任务，如润色某章一段 / 审摘要 / 改案例讨论】。
```
